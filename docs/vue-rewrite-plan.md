# Vue 全量迁移执行清单（蓝绿）

## 决策
- 采用蓝绿迁移：main(blue)稳定运行；vue-rewrite(green)全量重写后一次切流。
- 禁止在 main 中保留过渡代码；过渡仅存在于分支与环境层。

## 阶段
1. 基线冻结（blue）
- main 仅接受线上 bugfix，不再接收功能改动。
- 固定后端 API 契约与返回字段。

2. 全量迁移（green）
- 删除 legacy 入口依赖链（resources/js/main.js 仅保留最小兼容后最终移除）。
- Vue 接管模块：登录/大厅/练功/藏经阁/试炼/秘境/商城/排行/个人中心。
- Three.js 仅通过 scene bridge 调用，不允许组件直连底层对象。

3. 回归验收
- 路由刷新、登录恢复、全链路学习流程、商城、秘境计时、剧情进度持久化。
- 桌面/移动端可用性与层级检查。

4. 切流
- 反向代理或启动脚本切换到 green。
- 观察窗口内异常则回切 blue。

5. 收尾
- 合并 green -> main。
- 删除 legacy 代码与无用资源/样式。

## 验收门槛（必须全部通过）
- npm run build
- php artisan test（至少 API 相关）
- 关键手工链路：登录 -> 大厅 -> 练功 -> 试炼 -> 秘境 -> 商城 -> 退出
